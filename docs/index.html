What it does

From a Korean draft to a published Tistory post

The skill encodes the full pipeline: package the post, generate a fresh cover, reuse your browser session, and save privately.

Korean post → package

Converts a Korean Markdown or Hugo post into a ready-to-post Tistory package.

Auto cover prompt

Builds cover_prompt.md from the title, summary, tags, headings, and body signals.

Topic-specific cover

Requires a unique Codex Image 2.0 cover for every final post — no recycled banners.

Readable Tistory body

Adds clean spacing for paragraphs, headings, lists, tables, blockquotes, and code.

Reuse your Chrome

Drives the Playwright Agent CLI against an existing authenticated Chrome session over CDP.

Private by default

Refuses public publishing unless you explicitly confirm it in the request.

How it works

Six steps, one session

Writing comes first. Posting is the deterministic pipeline that runs after the article is ready.

  1. WriteDraft a Korean SEO post in Markdown or Hugo, following the Korean blog writing standard.
  2. Packageprepare_tistory_package.py builds the package and a topic-aware cover_prompt.md.
  3. CoverGenerate a unique cover with Codex Image 2.0 and save it as cover.codex.png.
  4. AttachConnect the Playwright CLI to your logged-in Chrome through the CDP endpoint.
  5. RenderUpload the cover and render readable Korean body HTML in the Tistory editor.
  6. PublishSave as 비공개 (private). Public conversion stays with the user, on explicit confirm.

Quickstart

Install and run

Link the skill into Claude Code or Codex, package a post, then publish privately.

1. Install (link the standalone clone)

# Claude Code
ln -s "$PWD" ~/.claude/skills/tistory-seo-blog-poster
# Codex
ln -s "$PWD" ~/.codex/skills/tistory-seo-blog-poster

2. Prepare a package

python3 scripts/prepare_tistory_package.py \
  --source /path/to/post.md \
  --output-dir /tmp/tistory-post

3. Publish privately (after generating the cover and attaching Chrome)

python3 scripts/tistory_playwright_cli.py \
  --package /tmp/tistory-post/package.json \
  --session tistory-cdp --attach none \
  --target-url "https://YOUR_BLOG.tistory.com/manage/newpost/?type=post" \
  --private-publish --confirm-private-publish

Guardrails

Safe by design

The skill is built to protect your account and keep covers authentic.

  • One cover per post. Never reuse a cover image; reject generic AI banners before upload.
  • Real covers only. No local fallback covers for final publishing when Codex Image 2.0 is available.
  • Your login stays yours. Tistory login lives in your browser — no cookies or profiles exported to the repo.
  • Private first. Private or draft flows by default; public publishing only on an explicit request.
